What is HTML?

HTML or HyperText Markup Languange is a technologie developed by Tim Berners-Lee at CERN in the early of the decade of 1990.

HTML is not a programming languange as was write in the name is a markup language!

Together with the HTML, Berners developed HTTP the protocol which allows computers comunicate via HTML and the first web browser.

In the actual days together with the HTML, others technologies as JavaScript and CSS compose what we see at the web pages.

Every HTML document have an <!DOCTYPE html> tag which tell the browser the type of document which will render.

Some tags:

  • <html></html> everything in the page goes inside this
  • <head></head> info about page goes here and the path for js script and css
  • <body></body> what goes be rendered goes here
  • <h1></h1> header, range 1 to 6
  • <title></title> give a name to the tab
  • <div></div> divider the HTML in sections
  • <ol></ol> ordered list
  • <ul></ul> unordered list
  • <li></li> list item
  • <p></p> paragraph
  • <a></a> anchor
  • <b></b> bold
  • <i></i> italic
  • <u></u> underline
  • <em></em> emphasis
  • <span></span> allow to set style to a part of the text inside a paragraph
  • <script></script> get language script or path to a file with script
  • <style></style> used to declare css to a document
  • <link> bind atributes of a css sheet
  • <img> insert image, dont have closure tag

<q></q> quotation mark.

Hierarchy: a HTML document is treat as elements tree, where each element can be a parent, a child, a brother, ancestors or descendents.

Hierarchy example:

in this example <div> are parent of <p> (child) and <p> is parent of <b> and <q>; <b> and <q> are brothers and descend from <div>.

DOM: Document Object Model are the interface to represent html documents read in the browser. After reading the document, the browser create a document representation object.

What is CSS?

CSS or Cascading Style Sheet is a mechanism to add a style to a markup document as HTML. Together with HTML and JavaScript is the cornerstone of the webpages technologie. Some of the benefits are: separation between content and formatting, easy and fast to modify, reuse, responsivity. A glood place to check out!; Even better to check out!

How it's work?

selector: define which element is goes to be affected; rule: define how the element will be affected; and contain properties and a value to each property; each property refers to a feature of the element. Example: 

                                
                                    selector {
                                        property1:value1;
                                        property2:value2;
                                        .
                                        .
                                        .
                                        property-n:value-n;
                                    }

A selector can be a HTML element or id or class, and there is the "*" selector which select all elements. Is possible too to combine diferent selector to create more rules. Still possible insert style directly into the html document, for this exists two possibilities, first is to insert the style directly to the html element, like: 
                                
                                    <tag style="property1:value1;property2:value2;...;property-n:value-n"></tag>
and the second is using the style tag in the head of the HTML document, like: 
                                
                                    <style>
                                        selector {
                                            property1:value1;
                                            property2:value2;
                                            .
                                            .
                                            .
                                            property-n:value-n;
                                        }
                                    </style>
From external documents, the style is inserted by a css document in the head of the html document using: 
                                
                                    <link rel="stylesheet" type="text/css" href="style.css">

In css the comments begin with "/*" and end in "*/".

CSS Box Model

Every HTML element is considered to be a box, every box contain a: margin, a border, a padding and a content. Boxes are separated in two types: inline or block. Following is an example how the boxes are defined. By clicking on the image you can read a W3School page more detailed about.

  • Margin: is the space between each box.
  • Border: is the boundary of the box.
  • Padding: is the space between the border and the content.
  • Content: is anything you want to display into the page.
Boxes that are block type generate a new line when used, because they occupy the whole horizontal space, they respect height and width properties. Inline type boxes don't create a new line and you size is defined by the content, the don't respect height and width properties; and is illegal to put a block type inside a inline type; boxes that are inline type margin and padding behave in a different manner.

Sizes

Boxes that ares block type can have the height defined. Height and WIdth can receive the following values:

  • Auto: browser will define the value.
  • number: a number in em, rem, px, etc.
  • initial: standard size.
  • inherit: same as parents elements.
max-height, max-width define the maximum size for the element, this will be the size shown if the interface is bigger than them, if the interface is lesser than them the element will be smaller. min-height, min-width define the minimum size for the element, if the content is smaller than them the size will be the size defined, other way there is no effect. Margin can receive the following values:
  • Auto: browser will define the value.
  • number: a number in em, rem, px, etc.
  • inherit: same as parents elements.
A margin can receive a negative number as value and is possible to give value to each side margin (top, bottom, left, right). Is possible to set each side margin as the following exemple: 
                                
                                    selector {
                                        margin: <value-top> <value-right> <value-bottom> <value-left>;
                                    }
                                    selector {
                                        margin: <value-top> <value-left-right> <value-bottom>;
                                    }
                                    selector {
                                        margin: <value-bottom-top> <value-left-right>;
                                    }
                                    selector {
                                        margin: <value-top-right-bottom-left>;
                                    }
                                    selector {
                                        margin-top:    value;
                                        margin-bottom: value;
                                        margin-left:   value;
                                        margin-right:  value;
                                    }
A bottom margin of an element will merge with the top margin of the below element and the bigger will be used as distance. The padding only differ from the margin by he won't accept the value auto or negative numbers. To define same border to all sides: 
                                
                                    selector {
                                        border: <size> <style> <color>
                                    }
Size: A number in em, rem, px, etc. Style: dotted, dashed, solid, double, groove, ridge, inset, outset, none, hidden Color: name, rgb, rgba, hex, hsl, hsla, etc. To define a different border to each side:

Display: Define the behavior of the element apresentation. Distinguish inline and block elements. Can modify element behavior, receive the values: none|block|inline|inline-block. None will show nothing (as if the element don't even exist), block will show the element as block, inline will show the elemente as inline, inline-block will show as inline but with the properties of a block. As none will show nothing, as if the element don't even exist then the element don't occupy any space, so there is another option to hide element and make them occupy some space, just using visibility wich receive the values: hidden|visible|collapse. A webpage is made of sections (header, navegation bar, extras, main content, footer). Some templates here! Position: elements are positioned by the properties: top, bottom, left right. This properties will work given a position, the property positiobn can receive the following values: static|relative|fixed|absolute|sticky.

  • Static is the standard behavior and ignore top, bottom, left, right properties. Usually used to undo a position given.
  • Relative, position the element relative to the original position and don't ignore top, bottom, left, right properties. Space left by the positioning are not gonna be filled.
  • Fixed, the position is fixed and it's mandatory at least one of the top, bottom, left, right properties. Wont let empty spaces and the element will use all the space independent the content size.
  • Absolute, the position is defined from the ancestors and it's mandatory at least one of the top, bottom, left, right properties. Wont let empty spaces and can overwrite other elements.
  • Sticky, the position is relative to the scrollbar and it's mandatory at least one of the top, bottom, left, right properties. Wont let empty spaces and can overwrite other elements.
